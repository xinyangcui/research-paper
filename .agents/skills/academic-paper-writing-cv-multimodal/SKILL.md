# Academic Paper Writing Skill for CV / Multimodal
# 学术论文写作技能：CV / 多模态方向

## 1. Purpose / 用途

This skill is for academic paper writing tasks, especially in:
- computer vision
- multimodal learning
- vision-language models
- document intelligence
- chart understanding
- table understanding
- OCR-related systems
- efficient high-resolution perception

本技能用于学术论文写作任务，尤其适用于：
- 计算机视觉
- 多模态学习
- 视觉语言模型
- 文档智能
- 图表理解
- 表格理解
- OCR 相关系统
- 高分辨率视觉理解与高效推理

Typical use cases / 典型用途：
- writing or polishing the abstract / 撰写或润色摘要
- writing the introduction / 撰写引言
- organizing related work / 组织相关工作
- describing methods rigorously / 严谨描述方法
- writing experiment sections / 撰写实验部分
- stating limitations honestly / 诚实撰写局限性
- refining paper language into academic style / 将内容改写为论文语体
- condensing notes into publishable prose / 将研究笔记压缩为论文表达

## 2. Core writing principles / 核心写作原则

### 2.1 Evidence-bounded claims / 结论必须受证据约束
Only write claims that are supported by:
- experiments
- ablations
- theoretical derivation
- implementation facts
- cited prior work
- user-provided material

只写那些能被以下内容支持的结论：
- 实验结果
- 消融实验
- 理论推导
- 实现事实
- 已有文献
- 用户提供的材料

If support is missing, do not present it as fact.

如果支撑不足，就不能把它写成既定事实。

### 2.2 Conservative academic tone / 学术表达必须克制
Use precise, restrained wording.
Avoid hype, slogans, and reviewer-irritating exaggeration.

使用精确、克制的措辞。
避免夸张、口号化表达，以及容易引起审稿人反感的吹捧式语言。

### 2.3 Separate fact, interpretation, and speculation / 区分事实、解释与推测
Always distinguish:
- what is directly shown
- what is a reasonable interpretation
- what remains uncertain

始终区分：
- 已被直接证明的内容
- 可以合理解释的内容
- 仍存在不确定性的内容

### 2.4 Explicit uncertainty / 明确承认不确定性
If something is unknown, ambiguous, or unsupported, state it directly.

如果某一点未知、模糊、或缺乏证据，必须直接说明。

Preferred wording / 推荐表述：
- “目前不清楚。”
- “根据现有结果无法确认。”
- “文中暂未提供足够证据支持这一判断。”
- “这一点仍需进一步验证。”
- “It is currently unclear.”
- “This cannot be confirmed from the available evidence.”
- “The current results do not provide sufficient support for this claim.”
- “This point still requires further validation.”

### 2.5 Write for reviewers, not for excitement / 为审稿人写，不为情绪写
The goal is clarity, credibility, and logical sufficiency.

写作目标是清楚、可信、逻辑充分，而不是“显得厉害”。

## 3. Global style requirements / 全局风格要求

### 3.1 Tone / 语气
Formal, neutral, precise, and restrained.

正式、中性、精确、克制。

### 3.2 Preferred sentence style / 推荐句式
Prefer:
- short-to-medium sentences
- explicit subject-object structure
- one main claim per sentence
- clear logical transitions

优先使用：
- 中短句
- 主谓宾清晰的句式
- 一句一个核心判断
- 明确的逻辑连接

### 3.3 Avoid / 避免
Avoid wording such as:
- “非常先进”
- “显著吊打”
- “完全解决了”
- “显然优于”
- “革命性”
- “obviously superior”
- “fully solves”
- “dramatically better” unless numerically justified
- “revolutionary”

避免使用：
- “非常先进”
- “显著吊打”
- “完全解决了”
- “显然优于”
- “革命性”
- “obviously superior”
- “fully solves”
- “dramatically better”（除非有明确数字支撑）
- “revolutionary”

### 3.4 Preferred wording / 推荐措辞
Use wording like:
- “We study the problem of...”
- “Our goal is to address...”
- “The main challenge lies in...”
- “We propose...”
- “The key idea is to...”
- “Experimental results show that...”
- “These results suggest that...”
- “However, this conclusion is limited to...”
- “该文研究……问题。”
- “本文旨在解决……”
- “该问题的主要挑战在于……”
- “本文提出……”
- “核心思路是……”
- “实验结果表明……”
- “这些结果说明……”
- “但这一结论仅限于……”

## 4. Section-specific writing rules / 各章节写作规则

## 4.1 Title / 标题
A good title should be:
- specific
- task-aware
- method-aware
- not overly promotional

好的标题应当：
- 具体
- 能体现任务
- 能体现方法
- 不夸张宣传

Preferred structure / 推荐结构：
- task + method + property
- problem + key idea
- efficient/high-resolution/document/chart/table + method cue

Examples of style / 风格示意：
- “Efficient Multimodal Reasoning for High-Resolution Documents”
- “Structure-Aware Chart Understanding with Hierarchical Visual Tokens”

Do not use vague titles like:
- “A Novel Framework for...”
- “An Effective Method for...”
unless there is no better concrete option.

避免使用过于空泛的标题：
- “A Novel Framework for...”
- “An Effective Method for...”

---

## 4.2 Abstract / 摘要
The abstract should typically contain:

摘要通常应包含以下五部分：

1. problem / 问题  
2. challenge / 难点  
3. method / 方法  
4. result / 结果  
5. significance or limitation scope / 意义或适用范围  

### Abstract writing rules / 摘要写作规则
- Start from the task, not from generic background.
- State the bottleneck concretely.
- Describe only the core method, not every module.
- Report key results conservatively.
- Avoid overclaiming generality.

- 从任务切入，不要从空泛背景开头。
- 具体说明瓶颈。
- 只写核心方法，不堆模块细节。
- 审慎汇报主要结果。
- 不夸大泛化性。

Preferred template / 推荐模板：
- “We study...”
- “Existing methods suffer from...”
- “To address this issue, we propose...”
- “Our method...”
- “Experiments on ... show that...”
- “These results suggest...”

---

## 4.3 Introduction / 引言
The introduction should answer:

引言必须回答以下问题：

1. What problem is studied? / 研究什么问题  
2. Why is it important? / 为什么重要  
3. Why are existing methods insufficient? / 为什么现有方法不够  
4. What is the proposed idea? / 本文做了什么  
5. Why might it work? / 为什么可能有效  
6. What do experiments show? / 实验说明了什么  
7. What are the contributions? / 贡献是什么  

### Introduction structure / 引言结构
Preferred 4-paragraph logic:

推荐四段式逻辑：

#### Paragraph 1: Background and problem / 背景与问题
State the task and practical or scientific importance.

说明任务及其实际/学术重要性。

#### Paragraph 2: Limitation of prior work / 现有方法不足
Be concrete:
- inefficient on high-resolution inputs
- weak structure modeling
- heavy OCR dependence
- poor long-context reasoning
- poor multilingual transfer

要具体，例如：
- 高分辨率下效率差
- 结构建模弱
- 过度依赖 OCR
- 长上下文推理差
- 多语种迁移差

#### Paragraph 3: Main idea / 本文核心思路
Describe the proposed solution compactly.

紧凑说明本文方案。

#### Paragraph 4: Contributions / 贡献
List only real contributions.

只列真实贡献。

### Contribution writing rules / 贡献写法规则
Each contribution should be one of:
- new task formulation
- new architecture
- new interaction mechanism
- new training objective
- new benchmark or dataset
- new efficiency-performance tradeoff
- strong empirical finding

每条贡献应属于以下之一：
- 新任务定义
- 新模型结构
- 新交互机制
- 新训练目标
- 新 benchmark 或数据集
- 新的效率-性能权衡
- 可靠的经验发现

Do not write fake contributions such as:
- “We conduct extensive experiments” alone
- “We provide useful analysis” alone
unless the analysis itself is genuinely substantial.

不要把以下内容单独冒充贡献：
- “我们做了大量实验”
- “我们进行了有用分析”

## 4.4 Related Work / 相关工作
The goal of related work is not to list papers mechanically.
Its goal is to position the paper.

相关工作的目的不是机械列文献，
而是明确本文所处的位置。

### Related work rules / 相关工作规则
- Organize by theme, not by publication year alone.
- Compare assumptions, methods, and limitations.
- End each subsection by pointing out what remains unresolved.
- Connect the gap back to your method.

- 按主题组织，不要只按年份罗列。
- 比较问题设定、方法和局限。
- 每小节结尾指出尚未解决的问题。
- 把研究空缺自然过渡到本文方法。

Preferred subsections for CV / multimodal / 推荐子节
- vision-language alignment methods
- OCR-based vs OCR-free document understanding
- chart/table structure modeling
- efficient high-resolution perception
- multimodal reasoning with long context

---

## 4.5 Method / 方法部分
The method section must prioritize:
- clarity
- modularity
- notation consistency
- causal logic

方法部分必须优先保证：
- 清晰
- 模块化
- 符号一致
- 因果逻辑清楚

### Method writing order / 方法撰写顺序
Use the following order by default:

默认按以下顺序写：

1. task formulation / 任务定义  
2. overall pipeline / 总体框架  
3. each module / 各个模块  
4. training objectives / 训练目标  
5. inference procedure / 推理流程  
6. complexity or efficiency discussion / 复杂度或效率分析  

### Method writing rules / 方法写作规则
- Define inputs and outputs explicitly.
- Introduce symbols before using them.
- Explain why each module exists.
- State whether a component is new or borrowed.
- If borrowed from prior work, say so directly.

- 明确定义输入输出。
- 符号先定义再使用。
- 解释每个模块存在的原因。
- 明确模块是新提出的还是沿用已有设计。
- 如果沿用前人方法，直接说明，不要伪装成原创。

### Preferred phrasing / 推荐表述
- “Given an input image ...”
- “We first encode ...”
- “The resulting tokens are then ...”
- “To preserve structural information, we ...”
- “Formally, ...”
- “The training objective consists of ...”
- “At inference time, ...”

---

## 4.6 Experiments / 实验部分
The experiment section should demonstrate:
- effectiveness
- fairness
- robustness
- interpretability when possible
- efficiency when relevant

实验部分需要说明：
- 方法有效
- 对比公平
- 结果稳健
- 尽量具备可解释性
- 如果相关，也要体现效率

### Experiment section structure / 实验部分结构
Recommended order:

推荐顺序：

1. setup / 实验设置  
2. main results / 主结果  
3. ablation / 消融  
4. efficiency analysis / 效率分析  
5. qualitative analysis / 定性分析  
6. failure cases / 失败案例  
7. discussion / 讨论  

### Experiment writing rules / 实验写作规则
- State datasets, metrics, and baselines clearly.
- Avoid cherry-picked interpretation.
- If gains are small, write them honestly.
- If fairness is limited, acknowledge it.
- If some settings differ from prior work, mention it.

- 清楚说明数据集、指标和 baseline。
- 避免挑结果说话。
- 如果提升很小，要诚实写。
- 如果公平性有限，要承认。
- 如果和前人设置不同，要明确说明。

### Preferred result phrasing / 结果表述推荐
- “Our method achieves better performance than ... under the reported setting.”
- “The gain is modest but consistent across ...”
- “The results suggest that ...”
- “However, the advantage becomes smaller on ...”
- “This indicates that ...”
- “在给定设定下，本文方法优于……”
- “该提升幅度不大，但在……上较为一致。”
- “结果表明……”
- “但在……场景下，该优势有所减弱。”
- “这说明……”

---

## 4.7 Limitations / 局限性
A good paper should state limitations proactively.

好的论文应主动说明局限。

### Limitation writing rules / 局限性写法
Discuss concrete limitations such as:
- dependence on OCR quality
- failure under dense layouts
- weak multilingual support
- limited long-range reasoning
- additional training cost
- higher inference latency
- dependence on synthetic data
- weak cross-domain generalization

可写的具体局限包括：
- 对 OCR 质量敏感
- 在密集版面下表现不稳
- 多语种支持不足
- 长距离推理仍弱
- 训练成本增加
- 推理延迟较高
- 依赖合成数据
- 跨域泛化有限

Avoid fake limitation statements like:
- “We leave more improvements to future work.”
unless tied to something specific.

避免空泛局限：
- “还有提升空间”
除非后面跟了具体问题。

---

## 4.8 Conclusion / 结论
The conclusion should:
- restate the problem
- summarize the idea
- summarize key evidence
- mention scope and limitation briefly

结论应当：
- 回到问题本身
- 概括方法
- 概括关键证据
- 简要提到适用范围与局限

Do not introduce major new claims in the conclusion.

不要在结论里新增重要主张。

## 5. Special rules for CV / Multimodal writing / CV与多模态论文特殊规则

### 5.1 Watch scale confounds / 注意规模混淆
When describing gains, always consider whether improvements may come from:
- larger backbone
- stronger OCR
- more pretraining data
- higher input resolution
- larger LLM
- more instruction tuning

描述性能提升时，必须考虑是否可能来自：
- 更大的 backbone
- 更强 OCR
- 更多预训练数据
- 更高输入分辨率
- 更大的 LLM
- 更多指令微调

Do not attribute all gains to your method unless supported.

如果没有证据，不要把所有提升都归因于方法本身。

### 5.2 Distinguish benchmark fit from general capability / 区分 benchmark 适配与普适能力
A result on one benchmark does not imply broad superiority.

单个 benchmark 上的结果，不等于广泛能力更强。

### 5.3 Document / chart / table tasks / 文档图表表格任务额外要求
When writing these papers, explicitly mention:
- structure modeling
- layout sensitivity
- OCR dependency
- high-resolution processing
- token budget or efficiency
- reasoning granularity

在写这些方向时，要显式提到：
- 结构建模
- 版面敏感性
- OCR 依赖
- 高分辨率处理
- token 预算或效率
- 推理粒度

## 6. Rewriting and polishing rules / 改写与润色规则

When rewriting user-provided content:

改写用户内容时：

### 6.1 Preserve meaning / 保持原意
Do not silently add unsupported claims.

不要暗中加入原文没有证据支持的新判断。

### 6.2 Improve structure / 优化结构
Make the text:
- clearer
- shorter where possible
- more academic
- logically tighter

使文本变得：
- 更清晰
- 尽量更简洁
- 更符合论文语体
- 逻辑更紧

### 6.3 Fix overclaiming / 修正夸大表达
If the original text is too strong, weaken it appropriately.

如果原文表述过强，应主动收敛措辞。

Example / 例子：
- “significantly outperforms all prior work”
-> “achieves improved performance over several representative baselines under the reported setting”

- “显著优于所有先前方法”
-> “在文中报告的设定下，相比若干代表性 baseline 取得了更优结果”

### 6.4 If evidence is missing / 若证据缺失
Replace unsupported certainty with cautious wording.

将无依据的确定表述替换为谨慎表达。

## 8. Forbidden behavior / 禁止行为

Never:
- fabricate numbers
- fabricate baselines
- fabricate citations
- fabricate ablation findings
- promise publishability
- present speculation as evidence
- overstate significance beyond experimental support

禁止：
- 编造数字
- 编造 baseline
- 编造引用
- 编造消融结论
- 承诺一定能发表
- 把猜测写成证据
- 超出实验支撑去夸大意义

---

## 9. Final rule / 最终规则

Your goal is not to sound grand.
Your goal is to write clearly, rigorously, and credibly.

你的目标不是写得“很大很强”，
而是写得清楚、严谨、可信。

When evidence is insufficient, say:

当证据不足时，请直接写：

**“目前不清楚。 / It is currently unclear.”**
